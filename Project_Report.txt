1. PROBLEM STATEMENT
--------------------
The objective of this project was to create a comprehensive product catalog where 
users can browse premium high-end items. The core requirement involved 
implementing a dynamic "Shopping Cart" using HTTP Sessions, allowing users to 
add, remove, and update items before proceeding to a final checkout. 

Apart from the basic requirements, this project has been developed as a fully 
functional, premium, production-ready e-commerce platform with advanced features,
professional aesthetics, and robust backend architecture.

2. THE SOLUTION & ARCHITECTURE
------------------------------
The solution is built using the Model-View-Controller (MVC) design pattern, 
ensuring a clean separation between data, business logic, and the user interface.

ARCHITECTURAL COMPONENTS:
- Model: JavaBeans (POJOs) representing Users, Products, CartItems, and Orders.
- View: JSP pages enhanced with JSTL (JavaServer Pages Standard Tag Library) 
  for dynamic content rendering without scriptlets.
- Controller: Java Servlets handling HTTP requests, managing session state, 
  and communicating with the DAO layer.
- Persistence: Data Access Object (DAO) pattern using optimized JDBC for 
  database interactions with MySQL.

TECH STACK:
- Backend: Java, Servlets, JSP
- Frontend: HTML5, CSS3 (Premium Dark/Glassmorphic design), Vanilla JS
- Database: MySQL 8.0
- Server: Apache Maven, Jetty/Tomcat

B. ADVANCED SESSION & COOKIE MANAGEMENT
- Sessions: Used to manage the persistent Shopping Cart across multiple requests.
- Cookies: Used for personalizing the experience (e.g., "Recently Viewed Products").

Snippet (CartServlet.java):
---------------------------
// Managing Shopping Cart in Session
HttpSession session = request.getSession();
List<CartItem> cart = (List<CartItem>) session.getAttribute("cart");
if (cart == null) {
    cart = new ArrayList<>();
    session.setAttribute("cart", cart);
}

// Personalization via Cookies
Cookie viewedCookie = new Cookie("last_viewed", String.valueOf(productId));
viewedCookie.setMaxAge(60 * 60 * 24); // 24 Hours
viewedCookie.setPath("/");
response.addCookie(viewedCookie);

C. CLEAN MVC & JSTL
Dynamic listing of products using JSTL <c:forEach> avoids cluttered Java code 
inside the frontend JSP files.

Snippet (products.jsp):
-----------------------
<c:forEach var="product" items="${products}">
    <div class="product-card">
        <img src="${product.primaryImageUrl}" alt="${product.name}">
        <h3>${product.name}</h3>
        <p>₹${product.price}</p>
        <button onclick="addToCart(${product.id})">Add to Cart</button>
    </div>
</c:forEach>


5. DATABASE SCRIPTS & SCHEMA
-----------------------------
The database is highly normalized across 9 primary tables.

SCHEMA SNAPSHOT:
- users: Manages customers and admins.
- products: Catalog details including pricing, stock, and ratings.
- categories: Product organization.
- orders/order_items: Transactional history with grand total, GST, and shipping.
- reviews: High-end customer feedback system.

CORE SCRIPT (frk_schema.sql):
----------------------------
CREATE TABLE products (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    name        VARCHAR(255) NOT NULL,
    price       DECIMAL(10,2) NOT NULL,
    stock       INT NOT NULL DEFAULT 0,
    is_featured BOOLEAN DEFAULT FALSE,
    FOREIGN KEY (category_id) REFERENCES categories(id)
) ENGINE=InnoDB;
